Overview

DIZ_WNDINFO is a structure used by DIZ_WINDOW to hold all necessary information for the creation of a window. Instances of this structure need not be created by the user- it is already a member of the DIZ_WINDOW class.

Declaration

//This is our window info structure- used to describe all the characteristics of a window we want
struct DIZ_WNDINFO {
	//The title for our window
	char *title;
	//The width, height, and colour bits for our window
	int width, height, bits;
	//The mode of our window- typically suggests 2D, 3D, or other relevant modes
	int mode;
	//Our Near and Far values- indicate the draw distances of our OpenGL scene
	GLfloat zNear, zFar;
	//Some flags to indicate whether we want a fullscreen window, a menu, etc.
	bool fullscreen, showCursor, menu;
	//A pointer to our window procedure for handling messages
	WNDPROC *wndProc;
};

char *title

This is simply the title for the window- what will appear at the top when created.

int width

The width in pixels of the window. Default is 640.

int height

The height in pixels of the window. Default is 480.

int bits

The colour bits for the window. Default is 16.

int mode

This is the OpenGL display mode of the window, so far only 2D or 3D. Set this equal to DIZ_2D or DIZ_3D for their respective drawing modes. Default is DIZ_3D.

GLfloat zNear

This is the nearest drawing point for our scene, i.e. anything closer than this point will not be drawn. Default is 0.1f.

GLfloat zFar

This is the furthest drawing point for the scene, i.e. anything beyond this point will not be drawn. Default is 100.0f.

bool fullscreen

Indicates whether or not the window shall be fullscreen or not. Default is false.

bool showCursor

Indicates whether or not the mouse cursor will be shown when inside the window. Default is true.

bool menu

Indicates whether or not the window is using a Menu. Currently however there is no way of implementing a menu in a DiZ Window. Default is false.

WNDPROC *wndProc

This must point to the window's callback procedure. This is an important part that should NOT be forgotten.

Example

The following example demonstrates the setup of a DIZ_WNDINFO structure. 
    //Forward-declare our Window Procedure function
    LRESULT CALLBACK WndProc(HWND hWnd, UINT uMsg, WPARAM wParam, LPARAM lParam);

    //Set our Window title
    window.info.title		= "Blind- LineForm Test";
    //Set our Window width and height
    window.info.width		= 640;
    window.info.height		= 480;
    //Specify our desired Colour Bits
    window.info.bits		= 16;
    //Disable fullscreen
    window.info.fullscreen	= false;
    //Disable the menu
    window.info.menu		= false;
    //Show the cursor
    window.info.showCursor	= true;
    //Set our Window for 3D drawing
    window.info.mode		= DIZ_3D;
    //Specify our Near and Far draw distances
    window.info.zNear		= 0.1f;
    window.info.zFar		= 100.0f;
    //Specify our wndProc function
    window.info.wndProc		= (WNDPROC *)WndProc;
When setting up a DIZ_WNDINFO structure, it is important you have your Window Procedure declared if not defined. This function will of course be used to handle all messages incoming to your window.

If you are not familiar with the WndProc function, you may look at the first full DiZ Example for further explanation.
